# Sales-Data-Analysis-with-Python-SQLite
This project demonstrates a complete mini-data pipeline: creating a local database, populating it with sales information, performing data aggregation using SQL, and visualizing the results with Python.

🚀 Overview
The goal of this task was to extract meaningful business insights (Total Quantity Sold and Total Revenue) from a raw sales dataset stored in a relational database.

🛠️ Tools Used
Python 3: The core programming language.

SQLite3: A lightweight, disk-based database that requires no separate server setup.

Pandas: Used for data manipulation and bridging the gap between SQL and Python.

Matplotlib: Used for generating the final visual bar chart.

📊 Data Workflow
1. Database Initialization
We initialized a file-based database named sales_data.db and created a sales table with the following schema:

id: Unique identifier for each transaction.

product: The name of the item sold.

quantity: Number of units sold.

price: The price per unit.

2. SQL Aggregation
To transform raw transaction rows into a summary, we used the GROUP BY clause. This allows us to "collapse" multiple sales of the same product into a single line of data.

The Query:

SQL
SELECT 
    product, 
    SUM(quantity) AS total_qty, 
    SUM(quantity * price) AS revenue 
FROM sales 
GROUP BY product;
3. Visualization
Using Pandas and Matplotlib, the aggregated revenue data was plotted into a bar chart (sales_chart.png) to provide a quick visual comparison of product performance.

📁 Files in this Repository
sales_analysis.py: The main Python script containing the database logic and plotting code.

sales_data.db: The SQLite database file generated by the script.

sales_chart.png: The output visualization showing Revenue per Product.
